# Next Feature Brief

Твоя задача — определить следующую разумную фичу для проекта и подготовить для нее `Brief` на основе текущего состояния проекта.

Сначала прочитай:
1. `AGENTS.md`
2. `CURRENT_PHASE.md`
3. `roadmap.md`
4. `memory-bank/index.md`
5. `memory-bank/features/README.md`
6. Релевантные уже существующие `FT-*`, если они нужны для понимания текущего baseline и change boundaries

Сфокусируйся на:
- текущем baseline проекта;
- том, какие эпики уже закрыты, какие частично закрыты, а какие еще не начаты;
- рекомендованной очереди работ из `CURRENT_PHASE.md`;
- зависимости между эпиками и фичами;
- выборе **одной следующей vertical-slice фичи**, а не большого эпика целиком.

Правила выбора следующей фичи:
- Не выбирай слишком крупную инициативу целиком, если ее можно нарезать на меньший slice.
- Учитывай уже реализованные фичи и не дублируй их scope.
- Следующая фича должна логично продолжать текущую фазу и опираться на существующий baseline.
- Если в `CURRENT_PHASE.md` есть явная рекомендация по следующему slice, считай ее приоритетной, если нет сильного конфликта с `roadmap.md`.
- Не предлагай сразу `implementation plan` или код. Нужен только `Brief`.

Требования к результату:
- Сначала коротко объясни, почему именно эта фича выбрана следующей.
- Затем подготовь полноценный `Brief` для этой фичи.
- `Brief` должен быть пригоден для дальнейшего ревью через `review-brief.md`.

Структура `Brief`:
1. Название фичи
2. Проблема
3. Стейкхолдер / пользователь
4. Контекст
5. Желаемый результат
6. Что не входит в цель

Требования к качеству `Brief`:
- Проблема должна быть конкретной и по возможности измеримой.
- Должен быть назван пользователь или стейкхолдер, для которого решается задача.
- Должно быть понятно, почему задача важна сейчас и как она следует из текущей фазы проекта.
- Не смешивай проблему с решением.
- Не используй двусмысленные формулировки вроде "удобно", "быстро", "при необходимости" без конкретизации.

Если данных недостаточно для однозначного `Brief`, сначала предложи самый вероятный вариант на основе `CURRENT_PHASE.md` и `roadmap.md`, а затем перечисли, какие вопросы стоит уточнить вручную.
